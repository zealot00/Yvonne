5. Key Findings

  • MockHSMBackend ≈ Software KEK: Both use AES-256-GCM, performance nearly identical
  • DEK-layer operations KEK-independent: EncryptVersioned/DecryptVersioned only use DEK
  • Real HSM expected 10-100x slower: PKCS#11 round-trip + chip processing
  • RaiseKey slowest (1.5ms): Transaction + version scan + DEK gen + wrap, but hourly only
  • Concurrent HSM 2.5x slower: MockHSMBackend mutex contention (real HSM: session pool)
