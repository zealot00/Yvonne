3. 结论

  • ScanPrefix：索引对前缀扫描有显著加速，数据量越大加速比越高
  • Put：索引带来微小写入开销（索引维护），但可忽略
  • Delete：通过主键索引定位，有索引/无索引差异极小（主键始终存在）
  • Get：走主键索引，无需对比（主键始终有索引）
