# Yvonne KMS 密评二级自评报告

> 版本：v1.3.0 | 日期：2026-06-30
> 评估标准：GB/T 39786-2021《信息系统密码应用基本要求》第二级
> 评估对象：Yvonne KMS v1.3.0（`-tags gmsm` 编译模式）

## 一、评估概述

### 1.1 评估目的

本报告对 Yvonne KMS 进行密码应用安全性自评，验证是否满足 GB/T 39786-2021 第二级密码应用基本要求，为正式密评提供基础材料。

### 1.2 评估范围

- 密码算法合规性
- 密钥管理安全性
- 密码应用方案完整性
- 网络与通信安全
- 审计与追溯能力

### 1.3 评估结论

| 类别 | 项数 | 合规 | 待改进 | 不合规 |
|---|---|---|---|---|
| 密码算法 | 5 | 4 | 1 | 0 |
| 密钥管理 | 8 | 8 | 0 | 0 |
| 密码应用方案 | 4 | 4 | 0 | 0 |
| 网络与通信 | 3 | 3 | 0 | 0 |
| 审计与追溯 | 4 | 4 | 0 | 0 |
| **总计** | **24** | **23** | **1** | **0** |

**综合结论**：23/24 项合规，1 项待改进（随机数发生器需接国密认证 RNG），整体满足密评二级要求。

## 二、详细评估

### 2.1 密码算法合规性

#### 2.1.1 对称加密算法

- **标准要求**：使用 SM4（GB/T 32907）
- **Yvonne 实现**：SM4-GCM 模式，`-tags gmsm` 编译启用
- **证据**：`internal/crypto/suite_gmsm.go` — `GMSMSuite.Cipher()` 返回 SM4-GCM
- **测试**：`internal/crypto/sm4_test.go` — SM4 KAT 向量验证
- **结论**：✅ 合规

#### 2.1.2 密码杂凑算法

- **标准要求**：使用 SM3（GB/T 32905）
- **Yvonne 实现**：SM3 + HMAC-SM3（审计哈希链）
- **证据**：`internal/audit/chain_gmsm.go` — `newHashChainSM3()`
- **测试**：审计链 HMAC-SM3 完整性验证
- **结论**：✅ 合规

#### 2.1.3 公钥密码算法

- **标准要求**：使用 SM2（GB/T 32918）
- **Yvonne 实现**：SM2 密钥对生成/加密/解密/签名/验签
- **证据**：`internal/crypto/sm2.go` — `GenerateSM2KeyPair/SM2Encrypt/SM2Decrypt/SM2Sign/SM2Verify`
- **测试**：`internal/crypto/sm2_test.go` — SM2 往返测试
- **结论**：✅ 合规

#### 2.1.4 随机数生成

- **标准要求**：使用经国家密码管理局批准的随机数发生器
- **Yvonne 实现**：`crypto/rand`（操作系统 CSPRNG）
- **证据**：`internal/memguard/entropy.go` — `GenerateSecureRandom()`
- **分析**：系统 CSPRNG 满足密码学安全要求，但未经国密认证
- **改进计划**：v2.0 接入硬件 RNG 模块（HSM 内置国密认证 RNG）
- **结论**：⚠️ 待改进

#### 2.1.5 算法禁用

- **标准要求**：严格模式下禁用非国密算法
- **Yvonne 实现**：`crypto.strict: true` 时仅允许 SM2/SM3/SM4
- **证据**：`internal/config/validator.go` — strict 模式校验
- **结论**：✅ 合规

### 2.2 密钥管理

#### 2.2.1 密钥生成

- **实现**：`memguard.NewSecureBufferFromRandom(32)` — CSPRNG 生成 256 位密钥
- **安全措施**：密钥生成后立即装入 SecureBuffer，明文不暴露
- **结论**：✅ 合规

#### 2.2.2 密钥存储

- **实现**：CMK 加密 DEK，密文存储于 PostgreSQL/BoltDB
- **安全措施**：明文密钥仅存在于 SecureBuffer 中，用完即 Wipe
- **结论**：✅ 合规

#### 2.2.3 密钥分发

- **实现**：版本化密文 `[version][nonce][ciphertext+tag]`，客户端自行解密
- **安全措施**：KMS 永不接触业务明文（信封加密模式）
- **结论**：✅ 合规

#### 2.2.4 密钥使用

- **实现**：RBAC + AllowedKeys 白名单 + 资源级授权
- **安全措施**：默认拒绝，通配符 + 前缀匹配
- **结论**：✅ 合规

#### 2.2.5 密钥更新

- **实现**：自动轮转（PostgreSQL Advisory Lock 选主 + 后台扫描）+ 手动轮转 API
- **安全措施**：旧版本 Deactivated，新版本 Active，向后兼容
- **结论**：✅ 合规

#### 2.2.6 密钥销毁

- **实现**：`clear()` + `runtime.KeepAlive()` + `ShredKey` 物理删除
- **安全措施**：Crypto-Shredding（删除加密密钥 = 销毁所有密文）
- **结论**：✅ 合规

#### 2.2.7 密钥备份

- **实现**：Shamir 秘密分割（K-of-N）+ 冷存储备份
- **安全措施**：分片存储于不同物理介质（USB）
- **结论**：✅ 合规

#### 2.2.8 密钥归档

- **实现**：SoftDelete 90 天 TTL + 审计日志记录
- **安全措施**：归档密钥可恢复，过期自动 Shred
- **结论**：✅ 合规

### 2.3 密码应用方案

#### 2.3.1 方案文档

- **结论**：✅ 合规

#### 2.3.2 密钥层次结构

- **实现**：CMK → KEK → DEK → 业务密文（三层信封加密）
- **结论**：✅ 合规

#### 2.3.3 密钥生命周期管理制度

- **文档**：`docs/compliance/key-lifecycle-management.md`
- **结论**：✅ 合规

#### 2.3.4 角色职责分离

- **文档**：`docs/compliance/role-separation-matrix.md`
- **实现**：管理员/审计员/业务/紧急操作员 RACI 矩阵
- **结论**：✅ 合规

### 2.4 网络与通信安全

#### 2.4.1 传输加密

- **实现**：TLS + RFC 8998 国密 TLS（SM2 双证书 + SM4 + SM3）
- **安全措施**：Cluster 模式强制 TLS，明文 gRPC 拒绝启动
- **结论**：✅ 合规

#### 2.4.2 完整性校验

- **实现**：HMAC-SM3 审计哈希链
- **安全措施**：每条日志链接前一条签名，篡改即断链
- **结论**：✅ 合规

#### 2.4.3 身份认证

- **实现**：JWT SM2 签名 + AppRole Token + K8s SA JWT
- **安全措施**：防算法混淆 + WithExpirationRequired + WithIssuer
- **结论**：✅ 合规

### 2.5 审计与追溯

#### 2.5.1 审计日志完整性

- **实现**：HMAC-SM3 哈希链 + anchor 文件恢复
- **安全措施**：anchor 损坏拒绝启动（不静默重置）
- **结论**：✅ 合规

#### 2.5.2 审计日志留存

- **实现**：可配置留存天数（默认 180 天）+ 自动清理
- **结论**：✅ 合规

#### 2.5.3 审计日志查询

- **实现**：`POST /api/v1/audit/query` + 分页 + 强制上限 10000
- **结论**：✅ 合规

#### 2.5.4 日志双写

- **实现**：文件 + Syslog 双写审计器
- **安全措施**：审计落盘失败 = panic 拒绝启动
- **结论**：✅ 合规

## 五、评估结论

Yvonne KMS v1.3.0 在 `-tags gmsm` 编译模式下，23/24 项满足 GB/T 39786-2021 第二级密码应用基本要求。唯一待改进项为随机数发生器（当前使用系统 CSPRNG，需接入国密认证 RNG 模块）。

**建议**：在正式密评前完成国密认证 RNG 接入，并委托国家密码管理局批准的测评机构进行正式评估。
