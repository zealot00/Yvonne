# 密钥生命周期管理制度

> 适用：Yvonne KMS v1.0+ | 日期：2026-06-26

## 1. 目的

规范 Yvonne KMS 中密钥的创建、使用、轮转、备份、销毁全生命周期管理，确保密钥安全可控。

## 2. 适用范围

本制度适用于所有通过 Yvonne KMS 管理的密钥，包括：
- Master Key（CMK）
- Data Encryption Key（DEK）
- 非对称密钥（RSA/ECDSA/SM2）
- 传输密钥（BYOK Transit Key）

## 3. 职责分工

| 角色 | 职责 | 人数要求 |
|---|---|---|
| 密钥管理员 | 创建/轮转/销毁密钥，保管 Shamir 分片 | ≥3 人 |
| 审计员 | 查询审计日志，定期审查密钥操作 | ≥1 人（与管理员分离） |
| 应用开发者 | 通过 SDK 调用加解密 API | - |
| 运维人员 | 部署/监控 Yvonne 实例，处理告警 | - |

## 4. 密钥创建

### 4.1 Master Key

- **生成方式**：CSPRNG 随机生成 256 位
- **存储方式**：Shamir 分片后分发给管理员（N=5, K=3）
- **审批要求**：至少 2 名管理员确认
- **记录要求**：审计日志记录生成时间、操作人

### 4.2 DEK

- **生成方式**：CSPRNG 随机生成（AES-256=32 字节 / SM4=16 字节）
- **存储方式**：CMK 加密后存入 PostgreSQL
- **审批要求**：业务方提交申请，管理员审批后创建
- **命名规范**：`{业务域}-{用途}`，如 `order-encrypt`、`payment-decrypt`

## 5. 密钥使用

### 5.1 加密

- 仅 Active 状态的密钥可加密
- 加密请求必须携带认证 Token
- `key_id` 必须在调用方 Policy 的 AllowedKeys 范围内

### 5.2 解密

- Active / Deactivated 状态的密钥可解密（向后兼容）
- SoftDeleted 状态的密钥仍可解密（回收期内）
- Destroyed 状态的密钥拒绝解密（永久不可恢复）

## 6. 密钥轮转

### 6.1 轮转周期

| 密钥类型 | 推荐周期 | 最大周期 |
|---|---|---|
| CMK | 12 个月 | 24 个月 |
| DEK（高敏感） | 30 天 | 90 天 |
| DEK（普通） | 90 天 | 365 天 |
| 传输密钥 | 单次使用 | 不留存 |

### 6.2 轮转流程

1. 管理员发起 `RotateKey` 或 RotationDaemon 自动触发
2. 旧版本 → Deactivated，新版本 → Active
3. 旧密文仍可解密（向后兼容）
4. 新加密使用新版本
5. 审计日志记录轮转时间、操作人、版本号

### 6.3 轮转验证

- 轮转后验证：旧密文可解密 + 新加密可用
- 监控告警：轮转失败 → Prometheus alert

## 7. 密钥备份与恢复

### 7.1 Master Key 备份

- Shamir 分片（5 份 3 门限）
- 每份分片写入独立 USB 介质
- 不同管理员分别保管
- 每年验证一次分片完整性

### 7.2 恢复流程

1. 收集 ≥3 份分片
2. 重启 Yvonne 进程
3. 通过 API 或 Admin UI 提交分片
4. 系统进入 Unsealed 状态
5. 审计日志记录恢复操作

### 7.3 紧急封印

- 触发条件：疑似泄露 / 安全事件 / 合规要求
- 操作：`POST /api/v1/sys/panic`（Admin Token + confirm）
- 效果：立即擦除所有内存密钥，不可逆
- 恢复：重启进程 + Shamir 解封

## 8. 密钥销毁

### 8.1 软删除（可恢复）

- 操作：`PATCH /api/v1/keys/{id}/soft-delete`
- 效果：标记 SoftDeleted，90 天后自动物理粉碎
- 恢复：90 天内可 `POST /api/v1/keys/{id}/restore`

### 8.2 物理粉碎（不可恢复）

- 操作：`DELETE /api/v1/keys/{id}/shred`
- 效果：`clear()` 内存 + `DELETE` DB 行 + 清空缓存
- 审批：至少 1 名管理员确认
- 后果：对应密文永久不可解密

### 8.3 销毁记录

- 审计日志记录：时间、操作人、key_id、版本
- 销毁后导出审计日志存档

## 9. 审计与监控

### 9.1 审计日志

- 所有密钥操作记录到审计链（HMAC 签名）
- 日志留存 ≥180 天
- 定期验证链完整性：`yvonne audit-verify --dir /var/log/yvonne`

### 9.2 监控指标

| 指标 | 告警阈值 |
|---|---|
| 密钥创建频率 | >10 次/小时（防滥用） |
| 解密失败率 | >5%（可能密钥状态异常） |
| 审计链验证失败 | 任何失败 |
| Vault Sealed | 任何非预期 Sealed |
| PG 不可用 | degraded 模式触发 |

## 10. 违规处理

- 未经授权创建/销毁密钥：立即撤销 Token + 审查
- 分片泄露：立即 Emergency Seal + 重新生成 CMK
- 审计链篡改：安全事件响应流程
