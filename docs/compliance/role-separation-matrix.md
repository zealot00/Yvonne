# 角色职责分离矩阵

> 适用：Yvonne KMS v1.0+ | 日期：2026-06-26

## 1. 职责分离原则

任何密钥操作必须满足**双人控制**或**审批留痕**，单一角色不能完成高风险操作的完整流程。

## 2. 角色定义

### 2.1 密钥管理员（Key Administrator）

| 维度 | 说明 |
|---|---|
| 职责 | 创建/轮转/销毁密钥，保管 Shamir 分片 |
| 认证 | AppRole Token（高权限） |
| 权限 | CreateKey / RotateKey / ShredKey / KeyOp |
| 禁止 | 审计日志查询（审计员职责） |
| 人数 | ≥3 人（满足 3 门限 Shamir） |

### 2.2 审计员（Auditor）

| 维度 | 说明 |
|---|---|
| 职责 | 查询审计日志，定期审查，验证链完整性 |
| 认证 | AppRole Token / JWT |
| 权限 | AuditQuery |
| 禁止 | 创建/轮转/销毁密钥（管理员职责） |
| 人数 | ≥1 人（必须与管理员不同人） |
| 独立性 | 审计员的 Token 由独立审批流程发放 |

### 2.3 业务服务（Application Service）

| 维度 | 说明 |
|---|---|
| 职责 | 通过 SDK 调用加解密 API |
| 认证 | AppRole Token / JWT / K8s SA JWT |
| 权限 | Encrypt / Decrypt（限授权的 key） |
| 禁止 | CreateKey / RotateKey / ShredKey / AuditQuery |
| 资源限制 | `AllowedKeys` 白名单（如 `order-*`） |

### 2.4 紧急操作员（Emergency Operator）

| 维度 | 说明 |
|---|---|
| 职责 | 紧急封印（Emergency Seal） |
| 认证 | Admin Token（独立于 AppRole/JWT） |
| 权限 | EmergencySeal |
| 禁止 | 日常密钥操作 |
| 人数 | ≥2 人（7×24 可联系） |

### 2.5 运维人员（DevOps）

| 维度 | 说明 |
|---|---|
| 职责 | 部署/监控/升级 Yvonne 实例 |
| 认证 | SSH / K8s kubectl（非 KMS API） |
| 权限 | 无 KMS API 权限（仅基础设施操作） |
| 禁止 | 直接访问数据库、查看明文密钥 |

## 3. 操作权限矩阵（RACI）

| 操作 | 管理员 | 审计员 | 业务服务 | 紧急操作员 | 运维 |
|---|---|---|---|---|---|
| 创建密钥 | **R/A** | I | - | - | - |
| 轮转密钥 | **R/A** | I | - | - | - |
| 软删除密钥 | **R/A** | I | - | - | - |
| 物理粉碎密钥 | **R** | A | - | - | - |
| 恢复软删除 | **R/A** | I | - | - | - |
| 加密数据 | - | - | **R/A** | - | - |
| 解密数据 | - | - | **R/A** | - | - |
| 查询审计日志 | - | **R/A** | - | - | - |
| 验证审计链 | - | **R/A** | - | - | - |
| 紧急封印 | - | - | - | **R/A** | - |
| Shamir 解封 | **R** | I | - | - | - |
| 备份分片 | **R/A** | I | - | - | - |
| 部署/升级 | - | - | - | - | **R/A** |

> R=Responsible（执行） A=Accountable（审批） I=Informed（知会）

## 4. 敏感操作审批流程

### 4.1 创建密钥

```
业务方申请 → 管理员审批 → 管理员执行 CreateKey → 审计员知会
```

### 4.2 物理粉碎密钥

```
管理员发起 → 审计员审批 → 管理员执行 ShredKey → 审计日志存档
```

### 4.3 紧急封印

```
紧急操作员判断 → 紧急操作员执行 EmergencySeal → 全员知会 → 启动恢复流程
```

### 4.4 Shamir 解封

```
≥3 名管理员提交分片 → 系统自动重组 → 审计日志记录 → 全员知会
```

## 5. Token 管理

### 5.1 Token 发放

| Token 类型 | 发放方式 | 审批 |
|---|---|---|
| 管理员 AppRole | 配置文件 + 环境变量 | 安全负责人审批 |
| 审计员 AppRole | 配置文件 | 独立于管理员审批 |
| 业务服务 AppRole | 配置文件 / K8s Secret | 业务负责人审批 |
| Admin Token | 配置文件 + 环境变量 | 安全负责人审批 |
| JWT | 外部 IdP 签发 | IdP 策略控制 |

### 5.2 Token 轮转

- AppRole Token：每 90 天轮转
- Admin Token：每 30 天轮转
- JWT：按 IdP 策略（通常 1-24 小时 TTL）

### 5.3 Token 撤销

- AppRole：修改配置文件 + 重启 / 热加载
- JWT：等待过期（无主动撤销，靠短 TTL）
- Admin Token：修改配置 + 重启

## 6. 合规检查清单

- [ ] 管理员与审计员角色分离（不同人）
- [ ] 紧急操作员 7×24 可联系
- [ ] Shamir 分片由 ≥3 名管理员分别保管
- [ ] AppRole Token 90 天内轮转
- [ ] 审计日志 180 天留存
- [ ] 审计链每月验证一次
- [ ] 密钥销毁有审批记录
- [ ] mTLS 在生产环境强制启用
